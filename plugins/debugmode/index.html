Plugin — Debugmode

How to install Debugmode

Clone Debugmode to your plugins dir (Source/client/plugins) in installed Source.js and run:

$ cd Source/client/plugins $ volo add sourcejs/Sourcejs_debug-mode $ cd Source $ grunt update

If you want to develop this plugin, read documentation.

Introducing to debugmode

Debugmode makes layout testing easier. It's a plugin for Source.js, and Debugmode can be used only with this tool. Debugmode is available on every specs page. It can be switched on/of by click on control on the left part of page over all its height.

Debugmode consists of two main parts.

The toolbar located within each section

The toolbar appears in each section when you hover over it. It allows you to check the layout on content stable without Web inspector and Firebug. To do this, enter the edit mode content, in addition you can revert the content of each section to the initial state. Also in the toolbar there is a link for switching-off debugmode.

The toolbar at the bottom of the page

This is the main toolbar that appears at the bottom of the page. For convenience, you can hide it without switching-off debugmode, as well as to change the height by simply pulling his resize control located on the middle of the toolbar. This toolbar contains several forms that help testing layout via the checklist.

For convenience, the main toolbar is divided into several forms.

Using debugmode, you can test the layout by looking problematic elements. Each element will be highlighted in red.

And when you hover over it, in the upper left corner of the page tooltip appears with information on why the element is considered to be problematic.

Problematic items can be selected using the form inside the toolbar:

  • Elements have lots of classes. When you point to an element in the error tooltip the number of classes of the element will be indicated.
  • Links with the wrong attributes.
  • Images with the wrong attributes.
  • Elements have lots of the number of nesting in DOM. When you point to an element in the error tooltip the level of nesting element relative section will be indicate.

The future of debugmode

In the near future we plan to add to debugmode a few more features:

  • grid over the layout for a more accurate test of indentation and layouts.
  • check for accessibility.
  • check the number of overdetermined styles for elements.
  • additional checks on the checklist, for example - the correct attributes of form fields.
