Grunt tasks

Intro

Grunt tasks are used to init, update and deploy Source engine instances, there is also some helpers for less compilation and watch command for comfortable development.

$ grunt init $ grunt update $ grunt updateDebug $ grunt deployServer $ grunt deployServerDebug $ grunt runWatch
  • init - is used for first Source setup both client and server side
  • update - copies all client side code to server root folder
  • updateDebug - copies files without minification
  • deployServer - task for minify and copy client side code to server by sftp
  • deployServerDebug - copies files to server without minification
  • runWatch - helper task for watching changes in client side code and copy them to public, also supports less

For Source install instructions please visit base documentation page.

Deploy to the server via sftp

For deploying to the server via sftp, you need to fill server access options in file /user/options/options.js. Also you need to fill ssh name/password in .ftppass. After these changes run:

$ grunt deployServer

All necessary Source.js files will be minimized, uglified and uploaded to your server.
